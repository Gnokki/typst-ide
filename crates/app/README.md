# crates/app (Tauri application shell)

This crate is the desktop application entry point. It bridges the web frontend (Monaco + JS) with the business logic exposed by [`crates/core`](../core).

## Role

- Initializes the Tauri window and serves `frontend/dist` as the UI
- Exposes **Tauri commands** callable from JS via `invoke()`
- Manages shared application state (SQLite connection, etc.)
- Contains **no compilation logic** everything goes through `typst-ide-core` (which is [`crates/core/src/compiler`](../core/src/compiler/))

## Exposed Commands

### Compilation / Preview

| Command | Return | Description |
|---|---|---|
| `render_preview(source)` | `Result<String, Vec<DiagnosticInfo>>` | Compiles Typst source to HTML (inline SVGs per page). Runs on a blocking thread pool to avoid freezing the UI. |
| `export_pdf(source, path)` | `Result<(), String>` | Compiles to PDF and writes the file to the given path. |
| `pick_pdf_path()` | `Option<String>` | Opens a native "Save As" dialog filtered to `.pdf`. |

### Project Management

| Command | Return | Description |
|---|---|---|
| `open_folder_dialog()` | `Option<String>` | Native folder picker dialog. |
| `create_project(name, base_path, content?)` | `Result<String, String>` | Creates a project directory with a `main.typ` file. |
| `open_project(dir_path)` | `Result<ProjectInfo, String>` | Reads the first `.typ` file found in the directory. |
| `save_file(path, content)` | `Result<(), String>` | Writes a file, creating missing parent directories. |

### Notes (SQLite)

| Command | Description |
|---|---|
| `add_note(title, content, scope, project_id?)` | Adds a note |
| `get_all_notes()` | Returns all notes |
| `get_global_notes()` | Returns notes not linked to any project |
| `get_project_notes(project_path)` | Returns notes linked to a project |
| `get_current_project_id(project_path)` | Hashes the project path into a stable ID |
| `delete_note(note_id)` | Deletes a note |
| `update_note(note_id, title, content, scope, project_id?)` | Updates a note |

### Utilities

| Command | Return | Description |
|---|---|---|
| `font_exists(name)` | `bool` | Checks whether a font family is available on the system. |

## Managed State (`tauri::State`)

| Struct | Inner type | Role |
|---|---|---|
| `NotesDbState` | `Mutex<rusqlite::Connection>` | SQLite connection shared across note commands |

The database is initialized on startup inside `app_data_dir()` (e.g. `~/.local/share/typst-ide/notes.db` on Linux).

## Structure

```
crates/app/
├── src/
│   └── main.rs          # Tauri commands + entry point
├── build.rs             # Generated by tauri-build
├── Cargo.toml
├── tauri.conf.json      # Tauri config (window, bundle, targets)
├── capabilities/
│   └── default.json     # Tauri v2 permissions
└── icons/               # Application icons
```

## Configuration (`tauri.conf.json`)

- **Frontend dist** : `../../frontend/dist`
- **Dev URL** : `http://localhost:1420`
- **Bundle targets** : `appimage` (Linux), `nsis` (Windows)
- **Identifier** : `com.typst.ide`

## Build Commands

```bash
# From crates/app/

# Development (with frontend hot-reload)
cargo tauri dev

# Linux release (AppImage)
NO_STRIP=1 cargo tauri build --target x86_64-unknown-linux-gnu

# Windows release (NSIS installer, cross-compiled from Linux)
cargo tauri build --target x86_64-pc-windows-gnu
```

> `NO_STRIP=1` is required on Linux because `linuxdeploy` does not support the `.relr.dyn` ELF sections produced by recent linkers.
